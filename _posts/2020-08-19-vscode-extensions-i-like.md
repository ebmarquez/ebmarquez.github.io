The other day I was working with a co-worker and explaining some details of the network configuration and he noticed some of my extensions.  Since he asked I went over the one I really liked any why I thought they were cool.  In my everyday job I work with a lot of JSON files and PowerShell.  As a result I work with VSCode as my primary IDE.  If you work with a lot of complex JSON files you will sympathize with the problem of handling the pesky curly braces.  To ease my everyday burden I use the following:

- [oderwat.indent-rainbow][indent-rainbow]  Indent rainbow is an extension that will color code tabs and spaces in the code your are viewing. For me I find this extension particular useful because i work with a lot of JSON and YAML files.  These files can be large and complex and this visual aid can be useful to determine if edits are landing in the correct location.  It can be configured for specific type of files, but I use it for all files.
- [CoenraadS.bracket-pair-colorizer][color-bracket]  Bracket Pair Colorizer like Indent Rainbow will color code the files you are viewing.  Unlike indent Rainbow where it will highlight the tabs and spaces in a file.  This will add colors to the start and end of a set of braces.   This can be handy when reviewing your code and identify if you have placed your braces in the correct location.  Its a small visual aid that will save your a few seconds when reviewing code.  A second and a second there save a lot of time in the long run.
- [aaron-bond.better-comments][better-comment]  Better Comments is a visual aid to help pop out actions that need follow up in your code.  At time you may have follow-ups that are not covered by the task you are working on.  It might be a how the code needs to be refactored, questions about the usage, or leaving a marker that shows this is deprecated code.  You can add a comment with TODO,  * Information, ! warning, and these keyword/charecters will be highlighted in your code.
